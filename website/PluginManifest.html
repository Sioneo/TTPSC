插件清单文件(plugin.manifest)

在1.8.15版本中，西奥小镇加入了插件清单文件（即plugin.manifest），这让外部插件也能在在线模式中使用。
此文件必须在插件文件夹的根目录（即第一个文件夹）中才能有效，且文件名必须为 plugin.manifest。
如果你想创建新加入的 .ttplugin 文件的话，插件清单文件也是必须要有的。
插件清单文件还有一个比较重要的作用就是使你的插件能够在外部（本地）插件列表中能够显示。

例子

以下是一个最简单、最基础的插件清单文件的例子：

plugin.manifest

{
  "id": "$example",
  "version": 1,
  "title": "标题",
  "text": "介绍",
  "author": "Sioneo",
  "thumbnail": "preview.png"
}

属性

插件清单文件支持以下属性：


  author

类型：字符串
本插件的作者。


  category

类型：布尔值
控制插件是否应该出现在外部（本地）插件列表中。

最低版本要求：1.11.72

此内容（category）需要最低1.11.72的游戏版本才能使用。


  id

类型：字符串
用于识别插件，所有id都应该是独一无二的。


  min version

类型：整数
插件可以使用的最小版本。


  multiplayer

类型：布尔值
控制插件能否在在线模式使用。

最低版本要求：1.8.33

此内容（multiplayer）需要最低1.8.33的游戏版本才能使用。


  once

类型：布尔值
控制插件清单是否应加载一次。
如果启用，当出现具有相同ID的另一个插件清单文件时，游戏将忽略它，不会报错。


  permanent

类型：布尔值
控制插件是否算作外部（本地）插件
