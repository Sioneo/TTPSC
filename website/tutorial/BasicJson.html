基础JSON内容

本页包括了JSON规范的一些内容。

注意！

西奥小镇的JSON解析器具有纠错能力，因此她可以将以字符串格式提供的值视为其他类型（例如将字符串格式的「true」视为布尔值） 但是，在某些情况下，由于属性接受不同类型的值，这可能会造成歧义。 因此，建议不要依赖西奥小镇的JSON解析器始终将字符串类型值视为其他类型，正确输入每一个值。
每一种数据类型的正确输入方式：
整数/浮点数（即为数字）：直接输入即可（例："level": 1）
布尔值（即true和false）：直接输入即可（例："active": true）
字符串（即非纯数字、非布尔值）：需用单/双引号括起（例："id": "$example"）

西奥小镇的JSON解析器还会在遇到不支持的属性（键）时会将其忽略。

基础JSON类型

以下包含了一些基础的JSON类型：

布尔值（Boolean）

布尔值只有两个值：真（true）或假（false），你可以简单地把她们理解为启用/停用。

例子

{
"active": true, // 设置启用某一内容。
"draw ground": false // 设置此建筑不绘制地面。
}


整数（Integer）

正如她的名字一样，可以是一个整数。

例子

{
"price": 2000 // 设置建筑的价钱为2000。
}


浮点数（Float）

浮点数表示一个小数。

例子

{
"pi": 3.1415926
}


字符串（String）

字符串表示一段文本，所有字符串都必须被单/双引号括起。

例子

{
"id": "$example",
"author": "Sioneo"
}

你可以使用转义字符\n在字符串中换行

{
"text": "Hello World!\nHello TheoTown!"
}


数组（Array）

数组用于为一个属性（键）提供多个值。

例子

{
"game": ["TheoTown", "西奥小镇"]
}


对象（Object）

对象用于存储属性。与数组相比，对象包含完整的一个或多个键值对，而数组只包含值。

例子

{
"array": ["TheoTown", "西奥小镇"],
"object": {"en": "TheoTown, "zh": "西奥小镇"}
}
